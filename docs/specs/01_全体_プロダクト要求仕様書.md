# 全体プロダクト要求仕様書（PRD）

## 1. 背景と目的
- 介護夜勤では誤報が多く、訪室空振りによる負荷が高い。
- 事故時/監査時に、客観的な時系列証跡が不足している。
- NightShiftOSは「トリアージ」と「証明可能な運用台帳」を同時提供する。

## 2. 対象ユーザー
- 夜勤介護士・看護師
- 夜勤責任者
- 施設管理者
- 将来: 保険会社・製薬/CRO（証明データの利用者）

## 3. 成功指標（MVP: 6か月）
- KPI-01: 重要アラート初動時間 P95 <= 60秒
- KPI-02: 誤報由来の空振り訪室率 30%相対削減
- KPI-03: 監査証跡自動生成率 >= 85%
- KPI-04: 夜勤申し送り作成工数 40%削減

## 4. 要求定義
| ID | 要求 | 優先度 | 受入条件 |
|---|---|---|---|
| PR-001 | 危険予兆（端座位/離床）をリアルタイム通知 | Must | Edge発生から通知までP95 3秒以内 |
| PR-002 | スタッフ入退室を自動記録 | Must | 入退室検知の記録欠落率1%未満 |
| PR-003 | オフライン/断線時も現場機能継続 | Must | Cloud断でもEdge検知+Appローカル記録継続 |
| PR-004 | 事故/監査に耐える監査証跡を保持 | Must | 誰が/いつ/どこで/何をしたか再構成可能 |
| PR-005 | 施設別設定（閾値/通知/役割）を変更可能 | Must | 変更操作から5分以内で反映 |
| PR-006 | 申し送り・日次レポートを自動生成 | Should | 指定期間を1クリックで出力 |
| PR-007 | センサー非依存の抽象化層を持つ | Must | 2種類以上のセンサー互換実装が可能 |
| PR-008 | 個人特定情報を最小化して扱う | Must | 画像/音声なし、暗号化/監査ログを実装 |
| PR-009 | 将来のRWD利用に必要な文脈を保持 | Should | Observation/Action/Deviationを保存 |

## 5. 非機能要件
- 可用性: Cloud制御面 99.9%/月
- 拡張性: 1施設 200室、同時接続スタッフ 100
- 性能: ダッシュボード更新遅延 <= 1秒（P95）
- 保全性: 監査ログ保管 5年（設定可）
- セキュリティ: RBAC + RLS + 監査ログ + 鍵ローテーション90日

## 6. MVPスコープ
### In
- 夜間見守り（端座位/離床）
- 入退室自動記録（Beacon）
- 通知/エスカレーション
- 監査証跡・PDFレポート

### Out（MVP）
- 全疾患向けデジタルエンドポイント自動計算
- 請求/レセプト連携
- 多言語運用

## 7. リスクと制約
- iOSバックグラウンド検知はOS仕様依存がある。
- 居室レイアウト差で閾値チューニングが必要。
- 施設ネットワーク品質の差が大きい。

## 8. トレーサビリティ起点
- PR -> F -> TC のマッピングを `90_トレーサビリティマトリクス.md` で正本管理する。
