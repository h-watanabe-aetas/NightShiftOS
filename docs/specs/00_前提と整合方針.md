# NightShiftOS 仕様前提と整合方針

## 1. 参照元
- `Aetas事業構想.docx`
- `NightShiftOSプロダクト要求仕様書（全体）` 配下の全DOCX

## 2. プロダクト定義（統一）
NightShiftOSは、介護施設の夜間見守りを対象に、以下を同時実現する運用OSである。
- 予兆検知とトリアージ（空振り訪室削減）
- 監査証跡付き自動記録（法的防衛・実地指導対応）
- 将来のRWD/DHT活用を見据えた証明可能データ基盤

## 5. 整合ルール
- すべての要求は、機能IDにマップされること。
- すべての機能は、実装単位（API/DB/モジュール/画面）にマップされること。
- すべての重要非機能要件は、計測可能なSLO/SLIを持つこと。
- すべての監査関連要件は、変更履歴・担当者・時刻・根拠データを保持すること。

## 6. 実装フェーズ（共通）
- Phase 1: 施設向け夜間見守りBPO価値（MVP）
- Phase 2: 運用台帳の高度化（品質保証/劣化検知）
- Phase 3: 証明可能データパッケージ化（RWD向け）
