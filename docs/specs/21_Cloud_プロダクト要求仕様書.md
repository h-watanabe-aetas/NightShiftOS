# Cloudプロダクト要求仕様書（PRD）

## 1. Cloudの使命
NightShiftOS Cloudは、Edge/Appの分散イベントを統合し、リアルタイム運用と監査証跡を提供する。

## 2. 要求定義
| ID | 要求 | 優先度 | 受入条件 |
|---|---|---|---|
| CR-001 | イベントを低遅延で取り込む | Must | API受付P95 200ms以下 |
| CR-002 | アラートを自動判定し通知 | Must | 危険イベントの通知生成率100% |
| CR-003 | 施設単位で厳格なデータ分離 | Must | RLS検証で越境取得0件 |
| CR-004 | 監査証跡の改ざん耐性 | Must | append-only + 署名ハッシュ |
| CR-005 | 設定変更履歴を完全保存 | Must | 誰がいつ何を変更したか追跡可能 |
| CR-006 | レポートを短時間出力 | Should | 24h分を30秒以内出力 |

## 3. 非機能
- 可用性99.9%
- RPO <= 5分、RTO <= 30分
- 監査ログ保管5年

## 4. 運用要件
- 監視: API, Queue, DB, Pushの4面監視
- 障害時: 自動エスカレーションRunbook
